# Strict Artifact Isolation

Keep the project root free from temporary caches, reporting output, and build artifacts generated by testing or linting tools.

1. **Centralized Caching:** All test-generated artifacts, test caches (like `.pytest_cache`), snapshots, linting caches (like `.ruff_cache` or `.eslintcache`), and reporting output (like `allure-results`) MUST be stored strictly within a dedicated `tests/` directory located *inside the current application or service workspace* (e.g., `apps/web/tests/.pytest_cache`, `apps/web/tests/allure-results`). You MUST NOT create a global `/tests` folder at the root of the monorepo to hold these artifacts.
2. **Tool Configuration:** When initializing or configuring new tools (linters, formatters, test runners), you MUST explicitly configure their output directories or cache locations to target the current workspace's `tests/` directory instead of their default root locations.
3. **.gitignore:** Ensure that any created cache directories within a workspace's `tests/` folder are immediately added to the `.gitignore` file so they are not accidentally committed.
